# FLSafeGuard 项目详细说明

## 1. 项目简介

FLSafeGuard 是一个面向竞赛展示场景的联邦学习安全攻防演示系统，核心目标是把“联邦学习中的投毒攻击与防御机制”做成一个可运行、可讲解、可交互、可对比的大屏展示项目。

项目当前版本强调以下几点：

- 能稳定运行，适合现场演示。
- 能直观展示节点状态、攻击效果和防御结果。
- 能通过前端控制面板快速切换攻击与防御。
- 能通过图表、拓扑图、节点详情和报告页完成完整叙事。

当前实现采用“轻量仿真引擎”来模拟联邦学习训练与攻击防御过程，优先保证演示稳定性和展示效果。后续如需进一步学术化，可将仿真模块替换为真实的 PyTorch + MNIST 训练流程。

## 3. 当前实现的核心能力

### 3.1 系统总览页

用于比赛演示时的主页面，包含：

- 当前训练轮次
- 全局准确率
- 全局损失
- 后门成功率
- 恶意节点数
- 隔离节点数
- 中央聚合器与 8 个客户端节点拓扑
- 实时事件流

### 3.2 攻击控制页

支持：

- 开始训练
- 暂停训练
- 手动推进下一轮
- 重置实验
- 切换标签翻转攻击
- 切换后门攻击
- 切换多节点协同投毒
- 调整恶意节点数
- 调整攻击强度

### 3.3 防御分析页

支持展示：

- 梯度异常分布图
- 节点相似度热力图
- 动态信誉评分折线图
- 聚合权重分配图
- 当前隔离节点列表

### 3.4 结果对比页

用于展示防御前后的差异，包含：

- 无防御时准确率
- 启用防御后准确率
- 无防御时损失曲线
- 启用防御后损失曲线
- 无防御时后门成功率
- 启用防御后后门成功率
- 恶意节点识别数

### 3.5 节点详情页

可查看单个节点的：

- 节点角色
- 当前状态
- 本地数据量
- 梯度范数
- 异常分数
- 信誉分
- 聚合权重
- 是否隔离
- 历史信誉变化
- 历史风险标记

### 3.6 实验报告页

支持展示：

- 实验编号
- 攻击类型
- 防御策略
- 防御前后准确率
- 防御前后后门成功率
- 被识别节点数
- 被隔离节点列表
- 自动生成摘要

并支持导出 Markdown 报告。

## 4. 后端仿真逻辑说明

当前版本不是使用真实深度学习长时间训练，而是使用仿真引擎来生成符合联邦学习安全演示逻辑的数据流。

### 4.1 客户端仿真

系统默认模拟 8 个联邦客户端：

- `client_1` 到 `client_8`
- 默认 2 个恶意节点
- 每个节点有自己的数据量、信誉分、异常分数、状态和聚合权重

### 4.2 攻击模拟

当前支持三类攻击：

1. 标签翻转攻击
2. 后门攻击
3. 多节点协同投毒

恶意节点在攻击启用后会生成偏离正常模式的“更新向量”，后端再基于这些更新生成可视化数据。

### 4.3 防御模拟

当前防御流程包含：

1. 异常检测
2. 相似度分析
3. 动态信誉评分
4. 基于信誉的聚合权重调整
5. 节点降权或隔离

### 4.4 数据记录

每轮结果都会写入 SQLite，包括：

- 轮次结果
- 客户端历史
- 系统事件日志

数据库位置：

- `backend/data/demo.db`

## 9. 演示建议流程

推荐按照下面顺序进行比赛展示：

1. 先打开系统总览页，说明这是联邦学习安全攻防大屏。
2. 点击开始训练，展示正常情况下模型精度逐步提升。
3. 切到攻击控制页，启用标签翻转攻击或后门攻击。
4. 返回总览页或防御分析页，展示恶意节点的异常和信誉变化。
5. 开启动态防御，观察节点被降权或隔离。
6. 打开结果对比页，讲解防御前后差异。
7. 打开报告页，展示实验摘要并导出报告。

## 11. 当前版本的限制

为了优先保证比赛演示效果，当前版本存在以下取舍：

- 当前训练流程是仿真驱动，不是完整真实 PyTorch 本地训练
- 后门攻击效果主要通过仿真指标展示，不是图像触发器真实训练结果
- 尚未加入实体硬件联动模块
- 尚未加入 PDF 导出，仅提供 Markdown 导出

这些限制不影响比赛中的展示逻辑，但如果后续需要论文级或工程级完整度，建议继续扩展。
