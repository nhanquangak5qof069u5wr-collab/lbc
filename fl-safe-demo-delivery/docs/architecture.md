# 系统架构说明

## 总体结构

系统采用前后端分离设计：

1. React 前端负责大屏展示、控制面板、图表和节点详情。
2. FastAPI 后端负责联邦训练仿真、攻击注入、防御评估和 WebSocket 推送。
3. SQLite 负责记录轮次结果、节点历史和事件日志。

## 后端数据流

1. 训练调度器驱动每一轮联邦训练。
2. 客户端仿真器生成正常或恶意更新。
3. 异常检测模块计算更新偏离程度。
4. 相似度分析模块生成余弦相似度热力图。
5. 动态信誉模块更新节点信誉分和聚合权重。
6. 鲁棒聚合模块压低恶意更新影响。
7. 轮次结果写入 SQLite，并通过 WebSocket 推送给前端。

## 前端展示流

1. 首次加载调用 `/api/system/status` 获取全量快照。
2. 页面建立 `/ws` WebSocket 订阅实时更新。
3. 控制按钮调用攻击、防御和训练控制接口。
4. ECharts 接收最新数据后实时重绘图表。

## 可扩展点

- 用真实 MNIST / Fashion-MNIST 数据替换当前轻量仿真引擎。
- 引入更完整的鲁棒聚合算法，如 Krum / Trimmed Mean。
- 增加物理节点灯光联动和串口控制模块。
